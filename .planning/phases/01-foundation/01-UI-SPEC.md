# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> React Native (Expo) project — shadcn is web-only and does not apply. Styling via plain `StyleSheet` as recommended in STACK.md.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — plain React Native `StyleSheet` |
| Preset | not applicable |
| Component library | none — custom components using React Native primitives |
| Icon library | `@expo/vector-icons` (Ionicons set) — bundled with Expo, zero extra install |
| Font | System default — SF Pro on iOS, Roboto on Android (no custom font load in v1) |

**Rationale (from STACK.md):** NativeWind v5 is pre-release against SDK 55; plain `StyleSheet` avoids the compatibility bet. Add NativeWind in v2 once pairing is confirmed.

## Spacing Scale

Declared values (multiples of 4, mapped to React Native `StyleSheet` constants):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4 | Icon gaps, tight inline padding |
| sm | 8 | Compact element spacing, button icon gap |
| md | 16 | Default padding for screen edges, between form fields |
| lg | 24 | Section separation, card internal padding |
| xl | 32 | Top padding below safe area on auth screens |
| 2xl | 48 | Illustration/icon top margin on empty states |
| 3xl | 64 | Vertical breathing room on onboarding screens |

Exceptions:
- Touch targets: minimum 44pt height on all tappable elements (iOS HIG requirement). Tab bar icon hit areas: 44pt × 44pt.
- Family code input field height: 56pt (larger than standard 44pt input — the code is the primary action on the join screen).

## Typography

All sizes in React Native `pt` units (equivalent to CSS `px` at 1:1 on logical pixels):

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16pt | 400 (regular) | 1.5 (24pt) | Form labels, input helper text, member names in lists |
| Label | 14pt | 400 (regular) | 1.4 (20pt) | Tab bar labels, secondary metadata, timestamps |
| Heading | 20pt | 600 (semibold) | 1.2 (24pt) | Screen titles, section headers, card titles |
| Display | 28pt | 600 (semibold) | 1.2 (34pt) | Onboarding hero text, welcome heading on Home tab |

Weights used: regular (400) and semibold (600) only. No bold (700), no light (300).

## Color

Mobile-appropriate palette. Dark-mode aware via React Native `useColorScheme` — provide both light and dark values.

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `#FFFFFF` | `#0F0F0F` | Screen backgrounds, modal sheets |
| Secondary (30%) | `#F5F5F5` | `#1C1C1E` | Cards, tab bar background, input fields, list rows |
| Accent (10%) | `#4F7BF7` | `#5E8CF8` | Primary CTA buttons, active tab indicator, family code text |
| Destructive | `#E53E3E` | `#FC8181` | Destructive actions only — no phase 1 usage (reserved for future) |
| Text primary | `#0F0F0F` | `#F5F5F5` | Body text, headings |
| Text secondary | `#6B6B6B` | `#8E8E93` | Labels, metadata, placeholder text |
| Border | `#E5E5E5` | `#2C2C2E` | Input outlines, dividers, card borders |

Accent (`#4F7BF7`) reserved for:
- "Create Family" primary button
- "Join Family" primary button
- "Continue" button on display name screen
- Active tab indicator dot/highlight in the bottom tab bar
- The rendered family code value on the share-code screen

Accent is NOT used for: passive text, list rows, section headers, nav title, or empty state illustrations.

## Layout Patterns

### Onboarding Screens (Welcome, Create, Join, Set Name)
- Full-screen scroll view with `contentContainerStyle` setting minimum height to fill screen
- Screen edge padding: 16pt left/right
- Primary CTA button: full width, 52pt height, 12pt border radius, accent fill, semibold 16pt white label
- Single text input per screen (except Create Family which has one input): 56pt height for code input, 48pt for name inputs, border `1pt solid #E5E5E5`, 8pt border radius, 16pt internal padding
- Keyboard behavior: `KeyboardAvoidingView` with `behavior="padding"` (iOS) / `behavior="height"` (Android)

### Share Code Screen
- Family code displayed in a bordered box, centered, 28pt semibold, accent color, letter spacing 4pt (for readability)
- Code format: 6-character alphanumeric (uppercase letters + digits, no ambiguous chars: 0/O, 1/I/L) — example: `FMLY42`
- "Copy Code" secondary action below the box: 14pt, accent color, no border

### Tab Shell
- 5 tabs: Home | Messages | Tasks | Location | Profile (source: CONTEXT.md D-05)
- Default tab on launch: Home (source: CONTEXT.md D-06)
- Tab bar uses `@expo/vector-icons` Ionicons: `home-outline`/`home`, `chatbubbles-outline`/`chatbubbles`, `checkmark-circle-outline`/`checkmark-circle`, `location-outline`/`location`, `person-outline`/`person`
- Active icon uses filled variant + accent color. Inactive uses outline variant + text secondary color.
- Tab labels: "Home", "Messages", "Tasks", "Location", "Profile"

### Home Tab
- Screen title: family name (20pt semibold, centered in nav bar)
- Member list: horizontal scroll of avatar circles (initials, 40pt diameter, accent background, white initials text)
- Recent activity section: shows "No activity yet" body copy below member list when empty
- No pull-to-refresh in phase 1 — static data display only

### Empty State Tabs (Messages, Tasks, Location)
- Centered layout: icon (48pt, text secondary color) + heading (20pt semibold) + body (16pt, text secondary) + no CTA
- Uses the screen's primary Ionicons icon as the empty state icon

### Profile Tab
- Simple read-only list: display name row + family name row
- No edit controls in phase 1
- "Sign Out" text button at bottom: 16pt, destructive color — present but shows a confirmation alert before signing out (only destructive action in phase 1)

## Interaction States

All interactive elements must implement all four states:

| Element | Default | Pressed | Disabled | Error |
|---------|---------|---------|----------|-------|
| Primary button | Accent fill, white label | 10% darker fill (`#4269D5` / `#4F7BF7`) | 50% opacity, no press feedback | Not applicable (buttons are not in error state) |
| Text input | Border `#E5E5E5` | Border accent | Border `#E5E5E5` | Border `#E53E3E` + error message below |
| Tab bar item | Outline icon, secondary text color | Brief scale 0.92 | Not applicable | Not applicable |
| Secondary text button | Accent color | 10% darker, underline | 50% opacity | Not applicable |

Loading state: primary button shows `ActivityIndicator` (white, size "small") replacing label text during async operations. Button is disabled during loading.
