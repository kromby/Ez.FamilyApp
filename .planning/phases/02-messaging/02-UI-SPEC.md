## Component Inventory

### Screen 1: Channel List (replaces messages.tsx)

File: `src/app/(tabs)/messages/index.tsx`

**Layout:**
- Full-screen FlashList of channel rows
- Header right: plus icon (Ionicons `add-outline`, size 24, color accent) — taps to open channel creation modal
- No section headers — single flat list sorted by last activity descending

**ChannelListItem component** (`src/components/messages/ChannelListItem.tsx`):
- Row height: auto, minimum 64px
- Left: `#` prefix + channel name (Body 16px, weight 400 for read; weight 700 for unread per D-04)
- Below channel name: last message snippet (Label 14px, textSecondary, 1 line, truncated with ellipsis)
- Right: timestamp of last message (Label 14px, textSecondary)
- Row separator: 1px border color line, full width
- Press state: surface color background flash (no persistent highlight)
- `#general` row: identical treatment to other channels — no special badge or lock icon

**Empty state (no channels — should not occur after D-03, but defensive):**
- Ionicons `chatbubbles-outline` size 48, textSecondary color, centered
- Heading: "No channels yet" (Heading 20px, textPrimary)
- Body: "Something went wrong. Pull down to refresh." (Body 16px, textSecondary, centered)

### Screen 2: Message Thread

File: `src/app/(tabs)/messages/[channelId].tsx`

**Layout:**
- FlashList inverted (newest at bottom) — messages render from bottom up
- Header title: channel name prefixed with `#` (Heading 20px, inherited from tab headerTitleStyle)
- Connection status banner: appears at top of message area only when SignalR is disconnected — 32px height, surface background, `Reconnecting...` in Label 14px textSecondary, centered. Hidden when connected.
- Message input row pinned at bottom above keyboard

**MessageBubble component** (`src/components/messages/MessageBubble.tsx`):

Flat layout (D-05 — no chat bubbles, Slack/Discord style):

- **First message in a sender group:**
  - Sender name (Label 14px, weight 600, textPrimary) on its own line
  - Timestamp (Label 14px, textSecondary) on same line as sender name, right-aligned or inline after name with Spacing.xs gap
  - Message text (Body 16px, weight 400, textPrimary) below sender row
  - Spacing.sm (8px) top margin from previous group

- **Subsequent messages in same sender group (within 5-minute threshold):**
  - No sender name repeat (D-07)
  - No timestamp repeat — timestamp is shown only on first message in group
  - Message text (Body 16px, weight 400, textPrimary) with Spacing.xs (4px) top margin from previous message in group

- **Sender grouping threshold:** Messages from the same sender within 5 minutes are grouped. A gap of 5+ minutes always starts a new group with sender name shown.

- **Failed message state (D-10):**
  - Red dot (8px circle, destructive color) left of the message text
  - Message text shows at normal opacity (not greyed out)
  - No retry UI in v1 — error dot is informational only

- **Optimistic message state (D-10):**
  - Renders immediately at normal opacity — no loading indicator
  - On API failure: error dot appears

**Reaction chips (D-14):**
- Appear below message text when reactions exist
- Each chip: emoji + space + count (Label 14px, textPrimary)
- Chip background: surface color; border: 1px border color; border-radius: 12px; padding: 2px 8px
- Current user's chip (user has reacted): accent color border instead of default border color
- Chips laid out in a horizontal row, wrapping if needed, Spacing.xs gap between chips
- Tap chip → toggle own reaction (add if not reacted, remove if already reacted)

**Long-press reaction picker (D-12, D-13):**
- Triggered by long-press (react-native-gesture-handler LongPressGestureHandler, 500ms threshold) on any message
- Reaction picker appears as an animated bottom sheet OR inline above the message (recommendation: inline above message using Reanimated, not a full bottom sheet — lighter for 6 fixed emojis)
- 6-emoji quick-pick row: 👍 ❤️ 😂 😮 😢 😡 (thumbs-up, heart, laugh, surprised, sad, angry)
- Each emoji button: 44px touch target, emoji rendered at 24px font size
- Picker background: surface color; border-radius: 22px; border: 1px border color; Spacing.sm horizontal padding
- Picker dismisses on selection or tap outside

**Message history loading (D-11 — Claude's discretion):**
- Batch size: 30 messages per page
- Pattern: infinite scroll upward — FlashList `onStartReached` triggers fetch of older page using cursor (last message ID)
- Loading indicator: ActivityIndicator (size small, accent color) pinned above the oldest visible message while fetching
- No "Load more" button — scroll-triggered only

**MessageInput component** (`src/components/messages/MessageInput.tsx`):
- Row height: 52px total (36px TextInput + 8px vertical padding each side)
- TextInput: multiline false for v1 (text-only per D-08); placeholder "Message #channelname"; Body 16px; textPrimary color; flex: 1
- Send button: Ionicons `send` size 20, accent color; 44px touch target; disabled (textSecondary color) when input is empty
- Row background: surface color; top border: 1px border color; horizontal padding: Spacing.md
- Keyboard avoiding: use `KeyboardAvoidingView` with behavior `padding` on iOS, `height` on Android

**Empty channel state (Claude's discretion — first time in a new channel):**
- Ionicons `chatbubble-outline` size 48, textSecondary, centered in message area
- Heading: "Start the conversation" (Heading 20px, textPrimary, centered)
- Body: "Send the first message in #channelname." (Body 16px, textSecondary, centered)

### Modal: Create Channel (D-02)

File: inline within `src/app/(tabs)/messages/index.tsx` using React Native Modal

**Layout:**
- Modal presentation: bottom sheet style (`transparent: true` modal + animated slide-up panel)
- Panel background: background color; border-radius: 16px top corners; Spacing.md padding
- Title: "New Channel" (Heading 20px, textPrimary)
- TextInput for channel name: Body 16px; placeholder "e.g. weekend-plans"; border: 1px border color; border-radius: 8px; padding: Spacing.sm Spacing.md; max length: 50 characters
- Validation: 1–50 characters, letters/numbers/spaces/hyphens only (regex: `/^[a-zA-Z0-9 -]{1,50}$/`)
- Inline validation error below input (Label 14px, destructive color): "Channel name must be 1–50 characters. Letters, numbers, spaces, and hyphens only."
- Confirm button: "Create Channel" — full width; accent background; white text; Heading 20px weight 600; height 48px; border-radius: 8px; disabled (surface background, textSecondary text) when input is empty or invalid
- Cancel: "Cancel" text button below confirm; Label 14px; textSecondary color; Spacing.md top margin; centered
- Dismiss: tap outside panel or Cancel button

## Interaction Contract

### Gestures

| Gesture | Target | Result |
|---------|--------|--------|
| Tap | ChannelListItem | Navigate to `[channelId]` screen |
| Tap | Plus icon in header | Open Create Channel modal |
| Tap | Send button | Submit message (disabled when empty) |
| Tap | Reaction chip | Toggle own reaction |
| Long-press (500ms) | MessageBubble | Show 6-emoji reaction picker |
| Tap | Emoji in picker | Add reaction + dismiss picker |
| Tap outside picker | Reaction picker | Dismiss picker |
| Scroll to top | Message thread | Load older messages (infinite scroll up) |
| Pull-to-refresh | Channel list | Refetch channel list |

### Navigation

| Action | Destination |
|--------|-------------|
| Tap channel | `src/app/(tabs)/messages/[channelId].tsx` |
| Back from channel | `src/app/(tabs)/messages/index.tsx` (Stack pop) |
| Cancel modal | Dismiss modal, stay on channel list |

### Keyboard

| Platform | KeyboardAvoidingView behavior |
|----------|------------------------------|
| iOS | `padding` |
| Android | `height` |

## Accessibility

- Send button: `accessibilityLabel="Send message"` even though it is icon-only
- Reaction chips: `accessibilityLabel="{emoji} reaction, {count} people"` + `accessibilityRole="button"`
- Emoji picker: `accessibilityLabel="React with {emoji}"` on each button
- Channel list items: `accessibilityRole="button"` on each row
- Failed message: `accessibilityLabel="Message failed to send"` on error dot
