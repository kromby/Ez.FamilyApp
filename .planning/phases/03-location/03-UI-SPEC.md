## Color

All values from `src/constants/theme.ts`. Use `useThemeColors()` — do not hardcode hex values.

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `#FFFFFF` | `#0F0F0F` | Screen background, modal background |
| Secondary (30%) | `#F5F5F5` | `#1C1C1E` | Member list row background, map card background, warning banner background |
| Accent (10%) | `#4F7BF7` | `#5E8CF8` | Allow button in permission modal, "Enable in settings" link text |
| Destructive | `#E53E3E` | `#FC8181` | Not used in this phase |

Accent reserved for:
1. The "Allow" / primary action button in the permission modal
2. The "Enable in Settings" tappable text in the Location tab warning banner

Do not apply accent to: member list rows, map pins, time labels, location pin icon in MessageBubble. Those use `textSecondary` or `textPrimary`.

Stale location visual treatment (D-05 — Claude's discretion):
- Location recorded < 1 hour ago: `textSecondary` color for time label, full opacity address.
- Location recorded 1–24 hours ago: `textSecondary` for both address and time label.
- Location recorded > 24 hours ago: `textSecondary` at `opacity: 0.6` for address line. Time label stays full `textSecondary`. No color changes — opacity fading only.

Map pin color: `accent` — uses the platform map's default callout, pin tint set to accent color.

## Interaction Contracts

### Permission modal (LOC-04)

- Shown once on first app open after auth, before any message is sent.
- State persisted in Zustand: `locationPermissionAsked: boolean`.
- "Allow" tap: calls `expo-location` `requestForegroundPermissionsAsync()`, dismisses modal.
- "Not now" tap: dismisses modal, sets `locationPermissionAsked: true`, no permission requested.
- Modal cannot be dismissed by tapping the backdrop.
- If system permission was already granted/denied (app reinstall case), skip showing modal.

### Location tab member list (LOC-02, LOC-03)

- Fetched via TanStack Query from new endpoint `GET /families/:familyId/members/locations`.
- All family members appear regardless of location data (D-02).
- Members with no location: show "No location yet" as address line, no time label.
- Tapping a member row with location data: expands `MapCard` inline below the row (Reanimated height animation, 250ms ease-out, to 200px). Tapping again collapses.
- Only one member expanded at a time — tapping a different row collapses the previous one.
- Map is not interactive during expand animation. Becomes interactive after animation completes.

### Warning banner (D-09)

- Shown at top of Location tab when current user's location sharing is off (profile toggle = off OR system permission denied).
- Two distinct cases communicated differently:
  - Profile toggle off: "Location sharing is off. Turn it on in your profile."  
  - System permission denied: "Location access is blocked. Enable it in your device Settings."
- "Profile" text links to profile tab (Expo Router `router.push('/(tabs)/profile')`).
- "Settings" text opens `Linking.openSettings()`.

### Location pin in MessageBubble (D-11)

- Rendered as `Ionicons location` at size 12, color `textSecondary`, with `marginLeft: Spacing.xs`.
- Appears inline at the end of the message text row, after the text content.
- Tapping opens a small modal (not inline expansion): shows member name, address, and time of that specific message's location. Title: "Sent from". Dismiss: tap backdrop or close button.
- Pin renders only when `message.latitude != null`.

### Profile location toggle (D-14)

- Positioned in a new "Privacy" section below the existing profile content.
- Section header: "Privacy" in `Typography.label` weight 600, `textSecondary` color, uppercase.
- Row height: 52px. Switch on right. Labels on left.
- On toggle off: no confirmation required. State saves immediately via `PATCH /users/me` with `{ shareLocation: false }`.
- Other members see "Location sharing off" instead of an address in the Location tab when this user has sharing disabled.
