Accent reserved for: Add button (enabled state), checkbox fill (checked state). Not used on metadata, borders, headers, or general interactive elements.

Completed task card: `textSecondary` color for task name + `textDecorationLine: 'line-through'` + card opacity 0.7. Active tasks: `textPrimary`.

Real-time highlight for tasks added by other family members: surface color with accent border-left (2px, accent color) — fades after 2 seconds via Reanimated opacity transition. Tasks added by the current user: no highlight.

Source: `src/constants/theme.ts` — Colors for both light and dark schemes.

## Component Inventory

### TaskCard

A card displayed in the flat list for each task.

```
[checkbox]  [task name — Body, textPrimary]
            [Added by {name} · {relative time} — Label, textSecondary]
            [Done by {name} — Label, textSecondary] (only if completed)
```

- Background: `colors.surface`
- Border radius: 8px
- Padding: `Spacing.md` horizontal, `Spacing.sm` vertical
- Min height: 44px
- No shadow/elevation — flat card consistent with MemberLocationRow pattern
- Bottom separator: 1px border in `colors.border` (list divider, not card shadow)
- Checkbox: Ionicons `checkmark-circle` (checked, accent color) / `ellipse-outline` (unchecked, `textSecondary`)
- Checkbox touch target: 44x44px minimum, `accessibilityRole="checkbox"`
- Swipe left: reveals red delete button (width 80px, background `colors.destructive`, label "Delete" in white Body weight 400)
- Completed state: task name gets `textDecorationLine: 'line-through'`, card opacity 0.7, `textSecondary` color
- Completed task sinks below active tasks with ~1000ms delay + slide-down animation (Reanimated layout transition)

### AddTaskBar

Pinned at bottom of screen above safe area inset. Always visible.

```
[TextInput — flex 1]  [Add button — Ionicons "add" + label "Add"]
```

- Background: `colors.surface`
- Top border: 1px `colors.border`
- Padding: `Spacing.sm` vertical, `Spacing.md` horizontal
- Input: Body (16px / 400), placeholder "New task…" in `textSecondary`
- Add button: accent background, white icon + label, border radius 8px, padding `Spacing.sm` horizontal
- Add button disabled: accent at 40% opacity, `accessibilityState={{ disabled: true }}`
- Max task name length: 200 characters (enforced client-side on TextInput `maxLength` prop)
- Auto-focus input when task list is empty (D-10); no auto-focus when tasks exist

### EmptyState

Centered vertically, shown when task list has no items.

```
[Ionicons "checkmark-circle-outline" — size 48, textSecondary]
[Heading: "No tasks yet"]
[Body: "Add tasks to keep your family on the same page."]
```

- Matches existing placeholder in `src/app/(tabs)/tasks.tsx` exactly — no visual change needed for empty state
- Icon top margin: `Spacing['2xl']` (48px) matching current implementation

## Interaction Contract

### Check-off / Uncheck (TASK-03)

1. User taps checkbox — optimistic toggle applied immediately in local state
2. PATCH request sent to API
3. On success: SignalR broadcast updates all other clients
4. On failure: revert local state, show inline error text below card ("Failed to update. Tap to retry.")
5. Tap again to uncheck — same flow, toggle back

### Add Task (TASK-02)

1. User types in AddTaskBar input
2. Add button enables when input has at least 1 non-whitespace character
3. Tap Add: optimistic item appended at top of list, input cleared immediately
4. POST request sent; on success SignalR broadcasts to family
5. On failure: remove optimistic item, restore input text, show error snackbar at bottom: "Couldn't add task. Try again."

### Delete Task (TASK-04)

1. User swipes left on a task card — delete button slides in (80px wide, destructive red)
2. Tap "Delete": task removed immediately from local list (optimistic), DELETE request sent
3. On success: SignalR broadcasts deletion to family
4. On failure (non-404): restore task in list, show error snackbar: "Couldn't delete task. Try again."
5. 404 response (already deleted by another member): silent ignore — task was already gone

### Real-time Sync (TASK-05)

- SignalR events: `TaskAdded`, `TaskUpdated`, `TaskDeleted`
- `TaskAdded` from another member: insert at top of list with 2px accent left border highlight, fade highlight after 2 seconds
- `TaskUpdated` from another member: update card in place, no highlight
- `TaskDeleted` from another member: remove card with brief fade-out (200ms opacity to 0)
- Pull-to-refresh available as fallback at all times

### Completed Task Ordering (D-02)

- Active tasks ordered newest first (most recently added at top)
- When task is checked: 1000ms delay, then slide-down animation moves card below all active tasks
- Completed tasks ordered among themselves newest-completed-first
- Unchecking a task moves it back to top of active section immediately

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Add" (button label in AddTaskBar) |
| Empty state heading | "No tasks yet" |
| Empty state body | "Add tasks to keep your family on the same page." |
| Error — add failed | "Couldn't add task. Try again." |
| Error — delete failed | "Couldn't delete task. Try again." |
| Error — toggle failed (inline) | "Failed to update. Tap to retry." |
| Delete swipe button | "Delete" |
| Input placeholder | "New task…" |
| Metadata — added attribution | "Added by {displayName} · {relative time}" |
| Metadata — completed attribution | "Done by {displayName}" |
| Screen header title | "Tasks" (Expo Router default tab label) |

Relative time format (consistent with Phase 3 location tab): "2h ago", "just now" (< 1 min), "1d ago".

Destructive action confirmation: No confirmation dialog — swipe-to-delete reveals button, single tap deletes. This is the standard mobile pattern (D-04). No alert/modal needed since delete is recoverable within the session via SignalR sync (another member can re-add).

Source: Empty state copy from existing `src/app/(tabs)/tasks.tsx` placeholder — preserved as-is.

## Accessibility

- All tappable elements: `accessibilityRole` declared (checkbox for task toggle, button for add/delete)
- Checkbox state: `accessibilityState={{ checked: task.completed }}`
- Add button disabled state: `accessibilityState={{ disabled: true }}`
- Swipe delete: also accessible via long-press context menu with "Delete" option (React Native Gesture Handler fallback)
- Minimum touch target: 44x44px on all interactive elements
